## ŁUK NARRACYJNY (każda sekcja = klatka filmu)
świat-problem (szufelka + chmura popiołu, BEZ produktu) → produkt WCHODZI jako rozwiązanie (stalowy
zbiornik zjada gorący popiół) → mechanizm z bliska (demo 1-2-3, filtr) → szerokość zastosowań
(garaż/auto/działka) → dowód (zestaw, porównanie, realne opinie, galeria) → wezwanie (mid-CTA) →
życie z produktem (porządek po paleniu, ciepły salon). Przejście „przed→po" = MIĘDZY sekcjami
(problem bez produktu → rozwiązanie z produktem), NIGDY w jednym kadrze.

## OSIE RÓŻNORODNOŚCI (matryca — minimum spełnione)
- **KONTEKSTY (≥3):** (1) dom z **kominkiem/kozą** (salon/kotłownia, popiół), (2) **garaż/warsztat**
  (beton, narzędzia, gruz/auto), (3) **na człowieku** (dłonie robocze / sylwetka niosąca za pałąk),
  (4) **plener/działka** (dmuchawa, liście). → 4 konteksty ✓
- **SKALE (≥3):** makro (filtr HEPA / klamra / włącznik) · średni (cały produkt w scenie) · szeroki
  lifestyle (garaż/salon w kadrze). ✓
- **ŚWIATŁA (≥2):** ciepłe wieczorne przy palenisku · chłodne dzienne betonowe w garażu/magazynie ·
  naturalne plenerowe. ✓
- **CZŁOWIEK ≥30%:** hero, problem, demo, zastosowania, mid-cta, final (6 z ~11 scen ≈ 55%). ✓
- **PERSPEKTYWY (≥2):** eye-level lifestyle · top-down/close makro · lekko od dołu (hero, bohater
  produktu). ✓

## OŚ POKRYCIE ZASTOSOWAŃ (z `MAPA-ZASTOSOWAN.md` — F1.7 egzekwuje szerokość, nie tylko scenografię)
Mapa niesie **≥2 RÓŻNE FUNKCJE** (sucho · mokro · nadmuch) ⇒ szerokość OBOWIĄZKOWA. Przewodnik pilnuje,
by każda z 3 funkcji miała NOŚNIK WIZUALNY (nie tylko etykietę w copy):
- **Ssanie na sucho** → hero (popiół) · rozwiązanie · demo · kafle kominek/pellet/gruz/warsztat-auto.
- **Ssanie na MOKRO (woda)** → **kafel „Woda i zalania — tryb mokro" (scena sc-zast-mokro)** + triada USP

  nie tylko wzmiankę.
- **Nadmuch (dmuchawa)** → kafel „Działka — dmuchawa na liście" + pas „zamiast wciągać — zdmuchuje" + FAQ.
- **Pokrycie ≥5 światów w mozaice** (kominek · pellet/kotłownia · gruz · warsztat/auto · **woda/mokro** ·
  działka). ⛔ ZAKAZ 4 kafle jednej funkcji (suche ssanie) — to zawężenie, które doktryna zamyka.

## REGUŁA RYTMU + ANTY-SZEW (strony obrazu zapisane OD RAZU)
Sąsiednie sceny nie dzielą kontekst+skala. Pełnokadrowe sceny (fullframe/typ B) sąsiadujące =
ZIG-ZAG L/P (na granicy zawsze foto↔papier). Sekcje kodowe/kontenerowane (zaufanie, zestaw,
porównanie, opinie, galeria, faq, zamow, demo-karta) IZOLUJĄ papierem → nie liczą się jako
pełnokadrowy sąsiad.

- **hero** — full-bleed (archetyp C, scena górą, karta nachodzi) — obraz PEŁNA SZEROKOŚĆ (top).
- `zaufanie` (kodowa, papier) — **separator** hero↔problem.
- **problem** — scena typu B, obraz **LEWO** / copy prawo.
- **rozwiazanie** — scena typu B, obraz **PRAWO** / copy lewo. *(zig-zag względem `problem` — jedyna
  para sąsiadujących pełnokadrowych; na granicy foto↔papier↔foto, nie foto↔foto).*
- **demo** — karta TOR-I na papierze (kontenerowana) — separator.
- **zastosowania** — scena typu B, obraz **LEWO** (sąsiad demo = kontenerowany, brak szwu).
- `zestaw`,`porownanie` (kodowe) — separatory.
- **mid-cta** — full-bleed scena, copy/CTA na scrimie; sąsiedzi kodowi (porównanie/opinie) izolują.
- `opinie`,`galeria`,`faq`,`zamow` (kodowe) — separatory.
- **final** — full-bleed scena; sąsiad `zamow` (kodowy) izoluje.

## CROSS-LANDING (≥3 z 5 osi vs BEZPOŚREDNIO poprzedni — mierzone i ocenione)
Poprzedni budowany (mtime): **home-zaradek** (+ mata, drapek w oknie N=3). Różnice:
1. **rodzina tła:** piasek/rozbielony beton ⟵ ≠ (home-zaradek chłodna/navy · mata krem-fiolet · drapek ciepły krem). ✓
2. **kolor akcentu:** #C2381B czerwień ⟵ ≠ #1F2A44 navy · ≠ #5E3A6E fiolet · ≠ #E0954A ochra (ΔE≥39). ✓
3. **font display:** Barlow Semi Condensed ⟵ ≠ Quicksand/Mulish · ≠ Fraunces · ≠ Baloo 2. ✓
4. **archetyp hero:** C (karta nachodząca) ⟵ ≠ poprzedni. ✓
5. **świat/materiał:** garaż/kominek/popiół, brudna gorąca robota, stal+czerwień ⟵ całkowicie inny świat. ✓
→ 5/5 osi RÓŻNE. „Inny produkt" nie liczono jako oś.

## HERO-STAGE (F1.7c — karta hero pod animację od makiety)
- **mechanika ruchu produktu (z PASZPORTU „MODEL UŻYCIA"):** **PASYWNY** — korpus statyczny, brak
  ruchomego mechanizmu widocznego z zewnątrz; sterowanie = JEDEN kołyskowy włącznik on/off. → nośnik
  ruchu = OTOCZENIE (popiół/sadza), produkt STATYKA.
- **nośnik ruchu:** DOMINUJĄCY FIZYCZNY OBIEKT = **wstęga/wir drobnego popiołu i sadzy** wciągana z
  paleniska do dyszy (sypkie/pył/dym) — wystawiona w kadrze, poza strefą tekstu, nieprzycięta krawędzią.
  ⛔ NIE „samo światło/cień/pyłki" (to tylko dodatek).
- **beat pętli (first=last):** wstęga popiołu spirala od paleniska w stronę dyszy i delikatnie wraca do
  klatki 1; produkt (stal + czerwona pokrywa) ostry, NIERUCHOMY (cechy wierności nie mogą się ruszać).
- **kadr pod i2v:** jeden plan, locked-off camera, fotorealizm; slot medium = CENTRALNY/DOMINUJĄCY punkt
  hero (desktop ≥~50% powierzchni, mobile pełnokadrowy), proporcje slotu = proporcje pętli; copy/CTA
  poza strefą ruchu. *(W TYM TEŚCIE wideo NIE generujemy — decyzja Tomka; makieta hero mimo to
  PROJEKTOWANA jako animowalna wg powyższego, żeby dała się ożywić bez przeprojektowania slotu.)*

## SCENY ANIMOWANE (ANIM-3 — hero + 2; rozrzut góra/dół; nie sąsiednie; nie TOR-I/zamow/faq)

NO product of ours in frame") + ref = styl-master, NIE packshot. ⛔ Nasz produkt NIGDY w scenie z
negatywną emocją.
